# تقرير النسخة النظيفة V56.3

## ما تم تنظيفه

- إزالة Cloudflare Worker القديم وملفات أصول المصدر المكررة من حزمة التسليم.
- إزالة أدوات الزرع والترحيل وإنشاء الأدمن القديمة؛ الإدارة الحالية تنفذ ذلك من Firebase ولوحة الصلاحيات.
- إزالة `firebase-tools` و`firebase-admin` من تبعيات جذر الواجهة؛ النشر يستخدم Firebase CLI العالمي، وFunctions تحتفظ بتبعياتها المستقلة.
- إزالة صورتي شعار PNG غير مستخدمتين من حزمة التسليم.
- إزالة ملفات التقارير وتعليمات الإصدارات القديمة والكاش والسجلات و`dist` و`node_modules` و`.git` والأسرار.
- تحميل خطوط WOFF2 فقط بدل نسخ WOFF+WOFF2، فصار بناء `dist` نحو 2.9MB.

## فحص الأمان

- `npm audit`: صفر ثغرات بعد تنظيف تبعيات الواجهة.
- `npm --prefix functions audit`: صفر ثغرات.
- لم يُعثر على Service Account أو Private Key أو Token سري داخل السورس.
- مفتاح Firebase Web وreCAPTCHA site key قيم عامة بطبيعتها، والحماية الفعلية في App Check وFunctions والقواعد.
- Firestore يمنع القراءة والكتابة المباشرة افتراضيًا، والعمليات الحساسة تمر عبر Functions.
- حذف حسابات العاملين والحركات المالية يتطلب أدمن ومصادقة حديثة ويسجل Activity Log.
- أضيف رفض بروتوكولات الروابط غير الآمنة في الأخبار والسوشيال، مع الإبقاء على HTTP/HTTPS فقط.
- CSP يسمح فقط بنطاقات Firebase/Google المطلوبة، ويمنع inline scripts وobject وframe ancestors.
- لم تُجر أي كتابة أو حذف في Firebase الإنتاج أثناء الفحص.
