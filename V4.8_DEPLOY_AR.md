# تشغيل ونشر إصدار V4.8.2

## ما تم داخل الكود

- فيديوهات الأخبار من YouTube وFacebook وTikTok أو رفع MP4/WebM حتى 30MB.
- صورة غلاف ومعاينة للفيديو من لوحة الإدارة، وتحميل الفيديو عند الضغط فقط.
- ربط الأخبار والصور والعروض بفرع طلخا أو المشاية أو كل الفروع.
- لوحة إدارة عربية وقوائم اختيار للفروع والتصنيفات بدل كتابة المعرّفات يدويًا.
- رابط SEO وخرائط الموقع مضبوط على Vercel، مع صفحة مستقلة لكل فرع.
- أحداث Analytics للحجز واختيار الفرع وإضافة خدمة والإلغاء.
- إدارة الحجز بالكود ورقم الهاتف، مع الإلغاء وطلب التعديل عبر واتساب.
- حد للمحاولات على الحجز والكوبونات والتقييمات وإدارة الحجز.
- دعم Firebase App Check وتجهيز إشعارات الحجز.
- كاش PWA جديد `v48` وسياسة حماية موحدة بين Firebase وVercel.
- فلتر الخدمات والعاملين في الإيرادات أصبح بقوائم عربية، مع ظهور الفرعين دائمًا.
- إزالة تحذيرات Firestore Rules غير المستخدمة.
- قسم العملاء يعرض اسم العميل ورقم الهاتف وآخر فرع وعدد الحجوزات وإجمالي المدفوع وآخر حجز.
- زر حذف نهائي للحجز ولعمليات الدفع والاسترداد، ظاهر للأدمن فقط ويطلب باسورد حساب الأدمن الحالي.
- الباسورد يُستخدم لإعادة التحقق مع Firebase Authentication ولا يُحفظ أو يُرسل إلى قاعدة البيانات.
- حذف الحجز يصحح إجماليات العميل والعامل، ويحذف سجل الدفع أو الاسترداد المرتبط ويفرغ الموعد.

## الاختبار محليًا

```bash
npm install
npm --prefix functions install
npm test
npm --prefix functions test
npm run build
npm run verify:build
```

## نشر Firebase Backend

```powershell
$env:FUNCTIONS_DISCOVERY_TIMEOUT="60"
firebase deploy --only "functions" --project el-mezaen-talkha
Remove-Item Env:FUNCTIONS_DISCOVERY_TIMEOUT
```

هذا الإصدار لا يحتاج إعادة نشر Firestore Rules أو Storage Rules. لا تحتاج أيضًا إلى نشر Firebase Hosting لأن الواجهة الأساسية على Vercel.

## رفع الواجهة إلى GitHub وVercel

```bash
git add .
git commit -m "feat: customers details and password protected deletion"
git push origin main
```

لو مشروع Vercel مربوط بالريبو، سيبدأ النشر تلقائيًا. أضف متغير البيئة التالي في Vercel إن لم يكن موجودًا:

```text
SITE_URL=https://el-mezaen-talkha.vercel.app
```

## تفعيل إشعارات الحجز

1. افتح Firebase Console ثم Project Settings ثم Cloud Messaging.
2. أنشئ Web Push certificate وانسخ الـVAPID Key العام.
3. ضعه في `window.__VAPID_KEY__` داخل `public/firebase-config.js`.
4. أعد رفع الواجهة، ثم افتح لوحة الإدارة واضغط زر الجرس واسمح بالإشعارات.

لا تضع أي Private Key داخل ملفات الواجهة.

## تفعيل App Check بدون تعطيل الموقع

1. أنشئ reCAPTCHA Enterprise provider من Firebase App Check وأضف نطاق Vercel والنطاق الرسمي.
2. ضع Site Key العام في `window.__APP_CHECK_SITE_KEY__` داخل `public/firebase-config.js`.
3. ارفع الواجهة وراقب وصول الطلبات الموثقة من شاشة App Check.
   عند الاختبار على localhost سيظهر Debug Token في Console؛ أضفه من App Check ثم Manage debug tokens.
4. بعد التأكد، أنشئ داخل مجلد `functions` الملف `.env.el-mezaen-talkha` وضع داخله:

```text
ENFORCE_APP_CHECK=true
```

5. أعد نشر Functions. لا تفعل enforcement قبل وضع Site Key ونشر الواجهة حتى لا يتوقف الحجز.

## Google Analytics وSearch Console

- Measurement ID موجود في إعداد Firebase ويُستخدم تلقائيًا لتسجيل أحداث الحجز.
- أضف `https://el-mezaen-talkha.vercel.app` إلى Google Search Console، ثم أرسل:
  `https://el-mezaen-talkha.vercel.app/sitemap.xml`
- عند شراء دومين رسمي، غيّر `SITE_URL` في Vercel و`window.__SITE_URL__` ثم أعد البناء والنشر.
