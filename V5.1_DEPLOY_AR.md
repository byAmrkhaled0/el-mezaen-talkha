# تشغيل ونشر إصدار V5.1 – الفلاتر ومشروبات الحجز

## الذي تم تنفيذه

- نقطة البيع تعرض قسمًا واحدًا فقط: خدمات، باقات، بضاعة أو مشروبات.
- عند اختيار الخدمات يلزم اختيار التصنيف، أو يمكن البحث باسم الخدمة مباشرة.
- شاشة الخدمات والتصنيفات لا تعرض كل الخدمات عند فتحها؛ اختر تصنيفًا أو استخدم البحث.
- زر «عرض الخدمات» داخل كل بطاقة تصنيف.
- الحجز يعرض اقتراحًا اختياريًا: «إيه رأيك تعدّل مزاجك بمشروب؟».
- المشروبات تأتي من أصناف المخزون التي نوعها `drink` وحسب الفرع المختار.
- الموقع العام يحصل فقط على الاسم وسعر البيع والتوفر؛ سعر التكلفة والبيانات الإدارية لا تخرج للعميل.
- الخادم يراجع سعر المشروب والفرع والرصيد، ثم يخصم الكمية داخل Firestore Transaction.
- عند رفض الحجز أو إلغائه ترجع المشروبات إلى المخزون مرة واحدة فقط.
- أكواد الخصم تُطبق على الخدمات والباقات، ولا تخصم من المشروبات.
- كاش التطبيق أصبح `v51` حتى لا تبقى النسخة القديمة على هواتف العملاء.

## إضافة المشروبات من لوحة الإدارة

1. افتح «البضاعة والمشروبات».
2. اضغط «إضافة صنف أو مشروب».
3. اختر النوع «مشروب» وحدد الفرع.
4. اكتب الاسم وسعر التكلفة وسعر البيع والرصيد الحالي، ثم فعّل الصنف.
5. يظهر المشروب في حجز الموقع لهذا الفرع طالما الرصيد أكبر من صفر.

## الاختبار قبل النشر

نفّذ داخل مجلد الريبو الحقيقي الذي يحتوي على مجلد `.git`:

```powershell
npm install
npm --prefix functions install
npm test
npm --prefix functions test
npm run build
npm run verify:build
node -e "import('./functions/src/index.js').then(m=>{console.log('Functions loaded:',Object.keys(m).length);process.exit(0)}).catch(e=>{console.error(e);process.exit(1)})"
```

## نشر Firebase

هذا الإصدار يحتاج نشر Functions لأن منطق كتالوج المشروبات وخصم المخزون موجود على الخادم:

```powershell
$env:FUNCTIONS_DISCOVERY_TIMEOUT="60"
firebase deploy --only "functions" --project el-mezaen-talkha
Remove-Item Env:FUNCTIONS_DISCOVERY_TIMEOUT
```

لا يحتاج Firebase Hosting لأن الواجهة منشورة على Vercel. ولا يحتاج تغيير Rules أو Indexes في هذا الإصدار.

## رفع GitHub وتشغيل Vercel تلقائيًا

```powershell
git status
git add .
git commit -m "feat: v5.1 filtered pos services and booking drinks"
git push origin main
```

بعد نجاح `git push` يبدأ Vercel النشر تلقائيًا من فرع `main`.

## لو النسخة المضغوطة ليست Git Repository

لا تنفّذ `git add` داخل مجلد النسخة المضغوطة. انسخ ملفاتها أولًا إلى مجلد الريبو الذي سبق استنساخه من GitHub، مع الحفاظ على مجلد `.git` الموجود فيه، ثم نفّذ أوامر الاختبار والنشر من مجلد الريبو.

## اختبار سريع بعد النشر

1. أضف مشروبًا برصيد 5 في فرع طلخا.
2. افتح الموقع واختر طلخا وخدمة، ثم افتح اقتراح المشروب وأضفه.
3. أكمل الحجز وتأكد أن الحجز ظهر في اللوحة وأن الرصيد أصبح 4.
4. ألغِ الحجز وتأكد أن الرصيد عاد إلى 5.
5. افتح نقطة البيع وجرّب الأقسام والتصنيف والبحث.
