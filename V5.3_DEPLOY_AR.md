# إصدار V5.3 – مشروبات مستقلة وإشعارات وSEO أسرع

## ما تغيّر

- المشروبات أصبحت مجموعة Firebase مستقلة باسم `drinks` ولا ترتبط بمخزون البضاعة أو تكلفة الشراء أو المصروفات.
- بيع المشروب فقط يُضاف إلى دخل المشروبات في الإيرادات.
- بيانات المشروب: اسم الصنف، النوع، الفرع، سعر البيع، واختيارات التحضير مفصولة بفواصل.
- المشروبات تظهر في اقتراح خطوة الحجز وفي نقطة البيع حسب الفرع، ويُحفظ اختيار التحضير داخل الحجز والشيك.
- تمت إضافة مفتاح Web Push العام وتشغيل استقبال إشعارات الحجز من Service Worker.
- تم تحسين شعار الواجهة من صورة كبيرة إلى WebP خفيفة، وتحسين الكاش والتحميل المؤجل للأقسام.
- تم تقوية عناوين ووصف وبيانات SEO المنظمة للصفحة الرئيسية والخدمات والفريق وصفحتي الفرعين.

## إضافة مشروب من لوحة الإدارة

افتح «المشروبات» ثم «إضافة مشروب» واكتب:

- اسم المشروب: `قهوة`
- النوع: `ساخن`
- الفرع: `طلخا` أو `المشاية`
- سعر البيع: مثال `25`
- اختيارات التحضير: `سادة، مظبوط، زيادة`

للمياه أو البيبسي اترك اختيارات التحضير فارغة. لو نفس المشروب متاح في الفرعين أضفه مرة لكل فرع، حتى يمكن تغيير السعر أو إيقافه في فرع واحد من غير التأثير على الآخر.

## الفحص قبل النشر

نفّذ داخل مجلد Git الحقيقي الذي يحتوي على مجلد `.git`:

```powershell
npm install
npm --prefix functions install
npm test
npm run test:functions
npm run verify:firebase
npm run build
npm run verify:build
node -e "import('./functions/src/index.js').then(m=>{console.log('Functions loaded:',Object.keys(m).length);process.exit(0)}).catch(e=>{console.error(e);process.exit(1)})"
```

## نشر Firebase

```powershell
$env:FUNCTIONS_DISCOVERY_TIMEOUT="60"
firebase deploy --only "functions,firestore:rules,firestore:indexes,storage" --project el-mezaen-talkha
Remove-Item Env:FUNCTIONS_DISCOVERY_TIMEOUT
```

لا تنشر Firebase Hosting؛ واجهة الموقع منشورة من GitHub إلى Vercel.

## اختبار حي بعد النشر

1. أضف «قهوة» لفرع طلخا بالخيارات `سادة، مظبوط، زيادة`.
2. افتح الموقع واختر طلخا وتأكد أن القهوة تظهر في اقتراح المشروب.
3. أضفها للحجز واختر «مظبوط»، ثم تأكد أن الحجز والشيك يعرضان الاختيار والسعر والكمية.
4. أكمل الدفع وتأكد أن المبلغ ظهر في «دخل المشروبات» ولم يتغير مخزون البضاعة ولم يُنشأ مصروف.
5. كرر الاختبار من نقطة البيع للفرع الآخر.
6. افتح لوحة الإدارة واضغط زر الجرس واسمح بالإشعارات، ثم أنشئ حجزًا تجريبيًا من هاتف آخر.

## الخطوة الأمنية المتبقية

مفتاح VAPID العام موجود بالفعل. يبقى App Check:

1. سجّل تطبيق الويب في Firebase App Check باستخدام reCAPTCHA Enterprise.
2. ضع Site Key العام داخل `window.__APP_CHECK_SITE_KEY__` في `public/firebase-config.js`.
3. اختبر الموقع أولًا مع `ENFORCE_APP_CHECK=false`.
4. بعد ظهور طلبات سليمة في لوحة App Check، فعّل `ENFORCE_APP_CHECK=true` داخل `functions/.env` وأعد نشر Functions.

لا تضع كلمة مرور أو مفتاحًا خاصًا داخل GitHub.

## ملاحظة SEO

الإصدار يجهز الموقع تقنيًا للأرشفة والسرعة، لكنه لا يضمن ترتيبًا أولًا فورًا. بعد النشر أضف الدومين إلى Google Search Console، أرسل `sitemap.xml`، وحافظ على محتوى محلي حقيقي وتقييمات وصور مضغوطة لكل فرع.
