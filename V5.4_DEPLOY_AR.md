# إصدار V5.4 – تحمّل الضغط، أوفلاين، وحسابات بفروع وصلاحيات

## ما تم تنفيذه

- رفع قدرة الدوال العامة إلى 80 طلبًا متزامنًا لكل نسخة وبحد أقصى 100 نسخة مع معاملات Firestore تمنع تضارب المواعيد والدفع.
- كاش للخدمات والباقات والمشروبات داخل Functions لمدة دقيقة لمنع تكرار قراءات Firestore وقت الزحام.
- منع موجة الطلبات المتطابقة داخل النسخة الواحدة عبر مشاركة عملية تحميل الكتالوج الجارية.
- حفظ آخر كتالوج ناجح على جهاز الزائر لمدة 7 أيام؛ يمكنه تصفح الموقع والسلة بدون إنترنت.
- الحجز والدفع والإلغاء لا يتم اعتمادهم أوفلاين حتى لا يتكرر حجز أو دفع. تظل اختيارات العميل محفوظة، ويؤكدها عند عودة الاتصال.
- Service Worker جديد `v55` لتشغيل الواجهة والصور والملفات المحفوظة أوفلاين.
- إعادة محاولة تلقائية للقراءات الآمنة في لوحة الإدارة عند الخطأ المؤقت، بدون إعادة تلقائية لعمليات الدفع أو الكتابة.
- إصلاح حظر التقييمات: البيانات الناقصة لا تُحسب، والحد الجديد 10 تقييمات في الساعة لكل هوية وزائر.
- إنشاء حساب عامل/مدير/استقبال/محاسب من لوحة الإدارة مع صلاحيات مستقلة لكل قسم.
- تعيين الحساب لفرع طلخا أو المشاية أو الفرعين. البيانات والإشعارات تُفلتر في الخادم حسب الفرع، والأدمن يرى الفرعين.
- العامل المخصص للمشاية لا يرى حجوزات طلخا ولا تصله إشعاراتها، والعكس.

## نشر Firebase

```powershell
npm install
npm --prefix functions install
npm test
npm run test:functions
npm run verify:firebase
npm run build
npm run verify:build

$env:FUNCTIONS_DISCOVERY_TIMEOUT="60"
firebase deploy --only "functions,firestore:rules,firestore:indexes,storage" --project el-mezaen-talkha
Remove-Item Env:FUNCTIONS_DISCOVERY_TIMEOUT
```

## تنظيف سجلات Rate Limit تلقائيًا

نفّذ مرة واحدة من Cloud Shell:

```bash
gcloud firestore fields ttls update expiresAt --collection-group=rateLimits --enable-ttl --project=el-mezaen-talkha
gcloud firestore fields ttls update expiresAt --collection-group=requestGuards --enable-ttl --project=el-mezaen-talkha
```

الحذف بواسطة TTL ليس فوريًا، لكنه يمنع تضخم سجلات الحماية بمرور الوقت.

## إبقاء الكتالوج دافئًا اختياريًا

الوضع الافتراضي لا يحجز نسخة دالة مدفوعة طوال الوقت. لو أردت تقليل أول تأخير بعد فترة خمول، أنشئ محليًا `functions/.env`:

```text
ENFORCE_APP_CHECK=false
KEEP_CATALOG_WARM=true
CATALOG_CACHE_MS=60000
```

ثم انشر `getCatalog`. تفعيل `KEEP_CATALOG_WARM` له تكلفة مستمرة صغيرة، لذلك لا تفعّله قبل ضبط Budget Alerts.

## إنشاء الحسابات

> مهم: أي حساب قديم غير الأدمن لازم تحدد له فرعًا وصلاحيات من جديد؛ الحساب غير المربوط بفرع لن تصله حجوزات أو إشعارات الفروع.

1. سجل الدخول بحساب الأدمن.
2. افتح «حسابات العاملين والصلاحيات».
3. اكتب الاسم والبريد والباسورد المؤقت.
4. اختر عامل أو مدير أو استقبال أو محاسب.
5. اختر طلخا أو المشاية أو الفرعين.
6. حدد الأقسام المسموحة ثم أنشئ الحساب.
7. يسجل صاحب الحساب من صفحة الدخول نفسها، ولا تظهر له الأقسام غير المسموحة.

## اختبار إلزامي بعد النشر

1. أنشئ عامل مشاية بصلاحيات نقطة البيع والحجوزات فقط.
2. سجل دخوله من نافذة خاصة وتأكد أن الإيرادات والرواتب والإعدادات غير ظاهرة.
3. أنشئ حجز طلخا وحجز مشاية، وتأكد أن حساب المشاية يرى الثاني فقط ويصله إشعاره فقط.
4. افصل الإنترنت بعد فتح الموقع وتأكد أن الخدمات والصور والسلة ما زالت ظاهرة.
5. جرّب تأكيد الحجز أوفلاين وتأكد أنه يطلب الاتصال ولا ينشئ حجزًا وهميًا.
6. أعد الإنترنت واضغط التأكيد مرة واحدة وتأكد أن كود حجز واحد فقط ظهر.
