# إصدار V5.2 – قسم المشروبات والحسابات وتجهيز Firebase

> هذا الدليل خاص بالإصدار القديم. في V5.3 أصبحت المشروبات مستقلة عن المخزون والمصروفات؛ استخدم `V5.3_DEPLOY_AR.md` للنشر الحالي.

## التقسيم الجديد

- «البضاعة والمخزون»: البضاعة ومستلزمات التشغيل فقط.
- «المشروبات»: قسم مستقل باسم المشروب وسعر التكلفة وسعر البيع والفرع والرصيد واختيارات التحضير.
- شراء البضاعة أو المشروبات يُسجل كمصروف ويُخصم من دخل المحل، وفي نفس العملية يزيد الرصيد.
- بيع المشروبات من الموقع أو نقطة البيع يدخل الإيرادات بعد تسجيل الدفع.
- تارجت العامل يُحسب من إيراد الخدمات فقط، وليس من المشروبات أو البضاعة.
- تظهر في اللوحة مؤشرات: دخل المشروبات، تكلفة شرائها، صافي المشروبات وقيمة رصيدها.

## أمثلة إعداد المشروبات

### قهوة

- الاسم: `قهوة`
- سعر البيع: السعر الذي يدفعه العميل.
- اختيارات التحضير: `سادة، مظبوط، زيادة`

### شاي

- الاسم: `شاي`
- اختيارات التحضير: `سادة، مظبوط، زيادة`

### بيبسي أو مياه

- اكتب الاسم والسعر واترك «اختيارات التحضير» فارغة.

الموقع يعرض الاسم والسعر وقائمة التحضير في خطوة مراجعة الحجز. الاختيار يُحفظ في Firebase ويظهر في لوحة الحجوزات والشيك.

## فحص المشروع قبل النشر

نفّذ من مجلد Git الحقيقي الذي يحتوي على `.git`:

```powershell
npm install
npm --prefix functions install
npm test
npm --prefix functions test
npm run verify:firebase
npm run build
npm run verify:build
node -e "import('./functions/src/index.js').then(m=>{console.log('Functions loaded:',Object.keys(m).length);process.exit(0)}).catch(e=>{console.error(e);process.exit(1)})"
```

## نشر Firebase كاملًا

```powershell
$env:FUNCTIONS_DISCOVERY_TIMEOUT="60"
firebase deploy --only "functions,firestore:rules,firestore:indexes,storage" --project el-mezaen-talkha
Remove-Item Env:FUNCTIONS_DISCOVERY_TIMEOUT
```

لا تنشر Firebase Hosting لأن واجهة الموقع مرتبطة بـVercel.

## إكمال إشعارات Firebase

1. افتح Firebase Console ثم Project settings ثم Cloud Messaging.
2. من Web Push certificates أنشئ Key Pair.
3. انسخ المفتاح العام فقط وضعه في `window.__VAPID_KEY__` داخل `public/firebase-config.js`.
4. ارفع GitHub وانتظر نشر Vercel، ثم افتح لوحة الإدارة واضغط زر الجرس واسمح بالإشعارات.

لا تضع أي مفتاح خاص أو باسورد داخل المشروع.

## إكمال App Check بأمان

1. من Firebase Console افتح App Check وسجل تطبيق الويب باستخدام reCAPTCHA Enterprise.
2. ضع Site Key العام في `window.__APP_CHECK_SITE_KEY__` داخل `public/firebase-config.js`.
3. ارفع الواجهة واختبر الحجز والدخول واللوحة أولًا بينما `ENFORCE_APP_CHECK=false`.
4. بعد ظهور طلبات سليمة في App Check، أنشئ محليًا `functions/.env` وضع داخله:

```text
ENFORCE_APP_CHECK=true
```

5. أعد نشر Functions فقط. ملف `.env` مستبعد من Git تلقائيًا.

## اختبار حي بعد النشر

1. أضف قهوة في فرع طلخا برصيد 5 واختيارات `سادة، مظبوط، زيادة`.
2. افتح الموقع واختر فرع طلخا وأضف قهوة مظبوط للحجز.
3. تأكد أن الحجز والشيك يعرضان «قهوة (مظبوط)» وأن الرصيد أصبح 4.
4. سجل الحجز «تم الدفع» وتأكد أن القيمة ظهرت في «دخل المشروبات».
5. ألغِ حجزًا غير مدفوع وتأكد أن الرصيد عاد مرة واحدة فقط.
6. من قسم المشروبات اضغط «تسجيل شراء مشروبات»، وسجل تكلفة وكمية؛ يجب أن تزيد الكمية وتظهر التكلفة كمصروف.
