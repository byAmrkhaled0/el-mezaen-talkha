# تشغيل ونشر إصدار V5.0 – إدارة المحل بالكامل

## المزايا الجديدة

- نقطة بيع داخل لوحة الإدارة لاختيار العميل والعامل والخدمات والبضاعة والمشروبات وطباعة الشيك.
- مخزون مستقل لكل فرع، مع سعر تكلفة وسعر بيع ورصيد وحد تنبيه للنقص.
- تسجيل شراء البضاعة يزيد المخزون، وبيعها من نقطة البيع ينقصه تلقائيًا.
- مصروفات الكهرباء والمياه والإيجار والصيانة والتسويق والمشتريات والرواتب تُخصم من الإيراد ويظهر صافي الربح.
- راتب أساسي وتارجت شهري ونسبة زيادة لكل عامل، مع حساب الراتب وتسجيل صرفه كمصروف مرة واحدة.
- التقييمات تُحفظ في Firebase بحالة انتظار، ويمكن قبولها أو رفضها أو تثبيتها والرد عليها من لوحة الإدارة.
- التقييمات المنشورة فقط تظهر في الموقع، مع متوسط حقيقي وعلامة للحجز الموثق.
- لوحة الإدارة قابلة للتثبيت كتطبيق مستقل على الهاتف واللاب، بأيقونة شعار مزين مصر.
- الصور الجديدة تُصغّر وتتحول إلى WebP تلقائيًا قبل رفعها، مع Lazy Loading ومقاسات محسنة للهاتف.
- كاش PWA جديد `v50` ومسارات الموقع ولوحة الإدارة تعمل من النسخة المبنية.

## طريقة إعداد العامل

من «فريق العمل» افتح العامل واكتب:

1. الراتب الأساسي الشهري.
2. تارجت الإيراد الشهري.
3. نسبة الزيادة عند تحقيق التارجت.

مثال: راتب 5000 جنيه، تارجت 20000 جنيه، زيادة 10%. إذا حقق العامل التارجت يصبح الراتب 5500 جنيه قبل أي تسوية شهرية.

## طريقة إعداد البضاعة والمشروبات

1. افتح «البضاعة والمشروبات» وأضف الصنف وحدد الفرع وسعر التكلفة والبيع.
2. اختر النوع «مشروب» للشاي والقهوة والمياه، أو «بضاعة للبيع» للمنتجات.
3. عند شراء كمية جديدة، سجلها من «المصروفات والمشتريات» واختر الصنف والكمية؛ الرصيد يزيد تلقائيًا.
4. عند إضافتها لشيك من نقطة البيع، الرصيد ينقص تلقائيًا.

## الاختبار قبل النشر

```powershell
npm ci
npm --prefix functions ci
npm test
npm --prefix functions test
npm run build
npm run verify:build
```

## نشر Firebase Functions

انشر الـFunctions أولًا حتى تكون نقطة البيع والمخزون والرواتب جاهزة قبل ظهور الواجهة الجديدة:

```powershell
$env:FUNCTIONS_DISCOVERY_TIMEOUT="60"
firebase deploy --only "functions" --project el-mezaen-talkha
Remove-Item Env:FUNCTIONS_DISCOVERY_TIMEOUT
```

هذا الإصدار لا يحتاج نشر Firebase Hosting ولا تغيير Firestore Rules أو Storage Rules.

## رفع GitHub وتشغيل Vercel

```powershell
git add .
git commit -m "feat: full shop management pos payroll inventory expenses and reviews"
git push origin main
```

إذا كان Vercel مربوطًا بفرع `main` فسيبدأ النشر تلقائيًا.

## التثبيت على الهاتف واللاب

- افتح `/admin/` من Chrome أو Edge بعد النشر وسجل الدخول.
- اضغط «تثبيت اللوحة» عند ظهوره، أو اختر Install app / Add to Home Screen من قائمة المتصفح.
- التطبيق يُثبت باسم «إدارة مزين» وبأيقونة شعار الموقع.

## إعدادات خارجية ما زالت تحتاج Firebase Console

- إشعارات الحجز تحتاج إضافة Web Push VAPID Key العام داخل `public/firebase-config.js`.
- App Check يحتاج إنشاء reCAPTCHA Enterprise Site Key وإضافته إلى الملف نفسه، ثم تفعيل `ENFORCE_APP_CHECK=true` بعد التأكد من وصول الطلبات.
- لا تضع أي مفتاح خاص أو باسورد داخل ملفات المشروع.
